## ConvertKit setup

This project reads ConvertKit settings from environment variables and exposes them to the templates as `convertkit_api_endpoint` and `convertkit_api_key`.

Files added:
- [.env.example](.env.example)

How to get the values
- ConvertKit form subscribe endpoint usually looks like `https://api.convertkit.com/v3/forms/<FORM_ID>/subscribe`.
- If you use a middleware or proxy, use that URL instead. Some setups require an API key â€” put it in `CONVERTKIT_API_KEY`.

Set environment variables (local)

Bash / macOS / WSL:

```bash
export CONVERTKIT_API_ENDPOINT="https://api.convertkit.com/v3/forms/<FORM_ID>/subscribe"
export CONVERTKIT_API_KEY="your_api_key_here"
```

PowerShell:

```powershell
$env:CONVERTKIT_API_ENDPOINT = 'https://api.convertkit.com/v3/forms/<FORM_ID>/subscribe'
$env:CONVERTKIT_API_KEY = 'your_api_key_here'
```

Test with curl (POST JSON):

```bash
curl -X POST "$CONVERTKIT_API_ENDPOINT" \
  -H "Content-Type: application/json" \
  -d '{"email":"you@example.com"}'
```

Deployment
- Set `CONVERTKIT_API_ENDPOINT` and `CONVERTKIT_API_KEY` in your hosting provider's environment variables (Netlify, Vercel, Docker, Kubernetes secrets, etc.).

Notes
- The templates will fallback to client-side behavior if these variables are empty. For Mailchimp use the `mailchimp_form_action` template var instead.
